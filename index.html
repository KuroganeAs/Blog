Latar Belakang

Selama mengerjakan App Reaction time, saya berusaha menciptakan aplikasi sederhana yang dapat mengukur waktu reaksi pengguna. Proyek ini saya bangun menggunakan HTML dan Node.js. Tujuan awalnya untuk membuat aplikasi yang ringan, interaktif, dan bisa dijalankan dengan mudah di berbagai sistem operasi. Namun, saya juga ingin belajar bagaimana membuat proses deployment menjadi lebih efisien dan profesional.

Pengalaman dan Tahapan Utama

Awalnya, saya menulis aplikasi seperti biasa di lingkungan lokal. Semuanya berjalan lancar di mesin saya, tapi kemudian saya mulai memikirkan bagaimana jika aplikasi ini harus dijalankan di perangkat atau sistem operasi lain. Di sinilah saya mulai mempelajari Docker.

Langkah pertama adalah membuat Dockerfile yang mendefinisikan bagaimana aplikasi saya dibangun dan dijalankan. Karena ini pertama kalinya saya menggunakan Docker, bagian ini menjadi tantangan tersendiri. Saya butuh waktu cukup lama untuk memahami cara menghubungkan aplikasi Node.js ke dalam kontainer Docker, terutama dalam mengatur port dan memastikan dependency berjalan dengan benar.

Namun, setelah berhasil, saya langsung bisa melihat manfaat besar dari Docker. Aplikasi saya bisa dijalankan hanya dengan satu perintah tanpa perlu instalasi tambahan. Bahkan ketika saya mencoba di sistem lain, hasilnya tetap sama — tidak ada error aneh akibat perbedaan lingkungan.

Tantangan yang Dihadapi

Tantangan utama saya tentu terletak pada tahap awal integrasi Docker. Saya perlu memahami bagaimana cara kerja layer dalam image, serta bagaimana mengatur EXPOSE port agar server Node.js bisa diakses dengan benar. Selain itu, saya juga harus memastikan semua dependency sudah tersalin ke dalam image, dan proses build tidak terlalu berat.

Saya sempat membandingkan performa antara menjalankan aplikasi langsung di sistem lokal dan melalui Docker. Hasilnya, Docker terasa sedikit lebih ringan dan terstruktur, meskipun pada proyek sekecil ini perbedaannya tidak begitu signifikan. Tapi saya bisa membayangkan betapa besar manfaatnya jika digunakan pada proyek yang kompleks.

Pelajaran dan Manfaat

Pengalaman pertama ini membuka pandangan baru bagi saya tentang bagaimana containerization bekerja. Docker tidak hanya membantu menjaga konsistensi lintas platform, tetapi juga menyederhanakan proses deployment. Saya mulai menyadari bahwa Docker sangat berguna untuk proyek besar, seperti signature forgery detection yang juga pernah saya kerjakan. Dalam proyek seperti itu, dependensi dan lingkungan kerja yang kompleks bisa dengan mudah dikelola menggunakan container.

Saya juga merasa debugging dan perbaikan kode menjadi lebih mudah karena Docker membuat lingkungan kerja lebih terkontrol dan mudah diatur ulang. Sekali saya menulis Dockerfile dengan benar, seluruh proses setup menjadi otomatis dan bebas repot.

Kesimpulan

Secara keseluruhan, pengalaman menggunakan Docker dalam proyek Reaction Time Test ini memberikan pelajaran berharga. Meskipun di awal cukup menantang, hasil akhirnya sepadan. Saya menjadi lebih paham tentang konsep isolasi lingkungan dan pentingnya menjaga keseragaman antara pengembangan dan deployment.

Docker membantu menjembatani perbedaan antar sistem operasi dan membuat aplikasi menjadi benar-benar lintas platform. Bagi saya, inilah poin pentingnya bahwa cross-platform deployment bukan hanya soal kemudahan menjalankan aplikasi di mana saja, tapi juga tentang efisiensi, keandalan, dan profesionalitas dalam pengembangan perangkat lunak.
